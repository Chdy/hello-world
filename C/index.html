二 使用闪回表应该注意哪些问题？
1. 整个 FLASHBACK TABLE 语句是在一个事务处理中执行的。可能会闪回所有指定表， 也可能不闪回任何指定表。
2. 在执行操作时，闪回表会获取对该语句指定的所有表的数据操纵语言 (DML) 锁。
3. 不会闪回受影响对象的统计信息。
4. 会保留所有现有索引。不会重新创建删除的索引。还会自动维护相关的提交时实体 化视图。
5. FLASHBACK TABLE 语句会写入到预警日志文件。
6. 只要不违反任何表约束条件，就闪回 FLASHBACK TABLE 语句中指定的表。如果在 执行闪回期间违反了任何约束条件，则会中止操作，此时表的状态与 FLASHBACK TABLE 语句调用之前的状态相同。 7. 不能将表闪回到早于执行数据定义语言 (DDL) 操作时的某个特定时间，因为执行数 据定义语言 (DDL) 操作会更改闪回操作涉及的表的结构或收缩此表。这种限制不适 用于只更改表存储属性的 DDL 语句。 8. 不能对系统表、远程表和固定表执行闪回表操作。 三 扩大数据库的方法有哪几种？ 1. 创建新表空间 2. 将数据文件添加到现有表空间 3. 增加数据文件的大小 4. 动态扩展数据文件

六 请描述表空间和数据文件的关系？
一个数据库可划分为多个逻辑存储单元，这些单元称为表空间，表空间可用于对相关逻辑
结构进行分组。每个数据库按逻辑都分为一个或多个表空间。可以为每个表空间显式创建一个
或多个数据文件，这样可在表空间中按物理方式存储所有逻辑结构的数据。
七 什么是方案？
方案是指由特定用户拥有的数据库对象的集合。对于生产数据库而言，这个用户代表的通
常不是一个人，而是一个应用程序。方案与拥有该方案的用户同名。方案对象是指直接引用数
据库数据的逻辑结构。方案对象包含表、视图和索引等多种结构。
八 Oracle内存结构包含哪些内容？
1. 系统全局区 (SGA):由所有服务器进程和后台进程共享
• 数据库缓冲区高速缓存 • 重做日志缓冲区 • 共享池 • 大型池 • Java 池 • Streams 池
2. 程序全局区 (PGA):专用于每一个服务器进程或后台进程。每一个进程使用一个 PGA
